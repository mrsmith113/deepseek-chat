# Видео-сценарий — День 26: Локальная LLM

**Продолжительность:** 5–7 минут  
**Формат:** экран + голос

---

## Вступление (0:00–0:40)

**Голос:**
> "Сегодня запускаем локальную LLM прямо на домашнем сервере. Никаких API-ключей, никакого облака — модель крутится у нас на машине. Используем Ollama и модель Qwen3 14B."

**На экране:** показываем терминал, вводим:
```bash
curl http://localhost:11434/api/tags
```
Видим JSON с `qwen3:14b` — модель загружена.

---

## Проверка через CLI (0:40–1:30)

**Голос:**
> "Сначала проверим через командную строку — это самый быстрый способ убедиться что модель работает."

**На экране:**
```bash
curl http://localhost:11434/api/generate \
  -d '{"model":"qwen3:14b","prompt":"2+2=?","think":false,"stream":false,"options":{"num_predict":10}}'
```
Видим ответ `"response":"4"` — всё работает.

---

## Запуск Python-скрипта (1:30–4:30)

**Голос:**
> "Теперь запустим наш скрипт — три запроса разной сложности. Первый простой — математика. Второй средний — объяснение концепции. Третий сложный — генерация кода."

**На экране:**
```bash
python main.py
```

Показываем как появляются ответы поочерёдно. Комментируем вслух:

**Запрос 1 (простой):**
> "Семнадцать умножить на шесть. Ответ пришёл за 5 секунд — быстро, потому что короткий."

**Запрос 2 (средний):**
> "Объяснение RAG. 130 токенов, 100 секунд. Аналогия с библиотекой — именно то что нужно для объяснения студентам."

**Запрос 3 (сложный):**
> "Просим написать код. 300 токенов. Модель пишет функцию с docstring, параметрами, примером — полноценный код."

---

## Вывод (4:30–5:30)

**Голос:**
> "Локальная LLM работает. Qwen3 14B — это 14 миллиардов параметров на нашем железе, без интернета. 
> Сейчас GPU занят другим процессом, поэтому модель крутится на CPU — медленнее, но работает.
> Когда GPU свободен — те же запросы занимают секунды, не минуты.
> Ключевой момент: через HTTP API — это значит любой наш скрипт, бот, агент может вызвать модель одним POST-запросом."

**На экране:** финальный вывод скрипта со всеми тремя ответами.
