# Объяснение — День 26: Локальная LLM

## Что сделали (одним абзацем)

Запустили языковую модель Qwen3 (14 миллиардов параметров) прямо на домашнем сервере через программу Ollama. Написали Python-скрипт, который обращается к модели через HTTP и задаёт ей три вопроса разной сложности — от простой математики до написания кода.

---

## Ключевые термины

### LLM (Large Language Model)
Большая языковая модель — программа, которая умеет генерировать текст. Представь очень умный автодополнение в телефоне, только в тысячи раз мощнее. Она обучена на огромном количестве текстов и умеет отвечать на вопросы, писать код, переводить.

### Локальная LLM
Это та же модель, но запущенная на ТВОЁМ компьютере, а не на серверах OpenAI или Google. Как разница между стримингом фильма (облачный ChatGPT) и просмотром с флешки (локальная LLM) — один требует интернет, второй работает без него.

### Ollama
Программа, которая упрощает запуск LLM локально. Как Docker для нейросетей: скачал модель, запустил, используй через HTTP. Одна команда `ollama run qwen3:14b` — и готово.

### HTTP API
Способ общаться с программой через сеть, как с сайтом. Мы отправляем POST-запрос на `localhost:11434` (наш собственный "мини-сайт" внутри компьютера), а модель отвечает JSON-ом с текстом.

| Аналогия | Техника |
|----------|---------|
| Ты звонишь в справочную | Отправляешь HTTP POST запрос |
| Оператор отвечает | Модель генерирует ответ |
| Трубка | Соединение через localhost:11434 |

### think:false
Qwen3 умеет "думать" перед ответом — генерировать внутренние размышления (chain-of-thought). Это делает ответы точнее, но медленнее. `think:false` — выключаем режим размышлений. Как разница между тем, кто сначала час обдумывает, и тем, кто отвечает сразу.

---

## Почему именно так

| Решение | Почему |
|---------|--------|
| Ollama, не llama.cpp напрямую | Проще в управлении, есть HTTP API из коробки |
| Qwen3 14B | Уже была на сервере, мощная, бесплатная |
| `think:false` | GPU занят другим процессом, на CPU думать слишком долго |
| `num_predict=300` | Баланс полноты ответа и скорости |
| `timeout=600` | На CPU генерация медленная (~0.18с/токен) |
