# 🐝 Hornest — Персонализированный мульти-агентный ассистент

**Task 13 — State Machine + Pipeline (рой агентов)**

---

## Архитектура

```
hornest.py       — точка входа, главный цикл CLI
agents.py        — 4 агента: Chat, Coder, Designer, Manager
pipeline.py      — рой агентов пайплайна (Planner→Executor→Validator→Summarizer)
task_state.py    — State Machine: этапы, сигналы [READY]/[PAUSE], параллельные субагенты
db.py            — SQLite: сессии, сообщения, память, профили, задачи
api.py           — DeepSeek API + подсчёт токенов с кэшем
memory.py        — три слоя памяти: Long-term / Working / Short-term
profile.py       — профили пользователей, мастер настройки
```

---

## Быстрый старт

### 1. Установи зависимости
```bash
pip install -r requirements.txt
```

### 2. Создай `.env` файл
```bash
copy .env.example .env
```
Открой `.env` и вставь ключ:
```
DEEPSEEK_API_KEY=sk-твой_ключ
```

### 3. Запусти
```bash
python hornest.py
```

При первом запуске:
- Запросит API ключ если нет в `.env`
- Запустит мастер настройки профиля
- Откроет Chat агент по умолчанию

## Пайплайн агентов (Manager)

Каждый этап задачи — отдельный специализированный агент:

```
👤 Пользователь
      ↓ описание задачи
🗓 Planner     → артефакт: ПЛАН
      ↓
⚙️ Executor    → артефакт: РЕЗУЛЬТАТ
      ↓
✅ Validator   → артефакт: ЗАКЛЮЧЕНИЕ
      ↓
🏁 Summarizer  → финальный итог
      ↓
👤 Готовый результат
```

Каждый агент получает артефакты всех предыдущих этапов.

## Команды

### Общие
```
/agent    — сменить агента
/profile  — управление профилями
/compare  — один вопрос × все агенты × два профиля
/memory   — все слои памяти
/remember — сохранить в долговременную память
/forget   — удалить из долговременной памяти
/working  — управление рабочей памятью
/tokens   — статистика токенов с разбивкой кэша
/sessions — сессии
/reset    — сбросить краткосрочную и рабочую память
/exit     — выход
```

### Manager — управление задачами
```
/task new <название>           — создать задачу (авто-переходы)
/task new <название> --confirm — с подтверждением каждого перехода
/task status                   — прогресс-бар задачи
/task next                     — принудительный переход на следующий этап
/task pause                    — поставить на паузу
/task resume                   — продолжить с того же места
/task rework                   — откатиться на предыдущий этап с комментарием
/task parallel                 — параллельные субагенты (демо)
/task swarm                    — настраиваемые параллельные субагенты
/task artifacts                — артефакты всех этапов
/task list                     — все задачи сессии
/task close                    — завершить задачу досрочно
```

## Ключевые механизмы

### Сигналы агентов
Агент вставляет маркер в конец ответа:
- `[READY]` — этап завершён, готов к передаче следующему агенту
- `[PAUSE]` — агент сомневается, нужна проверка пользователем

```
Режим авто (без --confirm):
  [READY] → автоматический переход к следующему агенту

Режим подтверждений (--confirm):
  [READY] → "Переходим к следующему агенту? [y/n]"
```

### Откат с доработкой (/task rework)
```
/task rework
→ выбираешь на какой этап откатиться
→ пишешь комментарий что переделать
→ агент получает весь предыдущий контекст + твой комментарий
→ дорабатывает точечно, не начинает с нуля
```

### Параллельные субагенты (/task swarm)
```
/task swarm
→ настраиваешь 2-4 субагента (имя, роль, вопрос)
→ все запускаются одновременно в отдельных потоках
→ результаты собираются и передаются оркестратору
→ оркестратор делает сводный вывод
```

### Три слоя памяти
```
🟢 Long-term  — глобальная, все сессии (профиль, решения, знания)
🟡 Working    — только текущая сессия (данные задачи)
🔵 Short-term — последние 6 пар сообщений
```

### Кэширование токенов
DeepSeek возвращает разбивку по токенам:
```
Новые токены:    $0.27 за 1M  (cache miss)
Кэш токены:      $0.07 за 1M  (cache hit, ~4x дешевле)
Выходные токены: $1.10 за 1M
```
Команда `/tokens` показывает экономию от кэша.
