# Объяснение — День 27: Интеграция локальной LLM в приложение

## Что сделали

Написали CLI-утилиту `ask.py`, которая принимает вопрос, ищет ответ в базе из 351 YouTube-видео и отвечает через Qwen3 14B — всё локально, без интернета.

**Аналогия из жизни:** Представь умного консультанта, у которого есть стопка ваших корпоративных документов. Ты задаёшь вопрос → он находит нужные страницы → читает их и формулирует ответ своими словами. Это и есть наше приложение.

---

## Каждый термин

**CLI-утилита** — программа, которую запускают в командной строке. Как `ping` или `ls`, только умнее. Не нужно открывать браузер — просто пишешь вопрос и получаешь ответ.

**RAG (Retrieval Augmented Generation)** — поиск + генерация. Сначала ищем нужные куски текста (Retrieval), потом передаём их модели (Augmented), и модель генерирует ответ (Generation). Без RAG модель отвечает из памяти — может ошибиться. С RAG — отвечает по конкретным документам.

**Qdrant** — база данных для векторного поиска. Обычная БД хранит текст и ищет по ключевым словам. Qdrant хранит векторы (числовые представления смысла) и ищет по смыслу. Пример: запрос "пошлина на гаджеты" найдёт документ про "таможенный сбор на электронику" — хотя слова разные.

**Qwen3 14B** — языковая модель на 14 миллиардов параметров. Работает локально через Ollama. "14B" = 14 billion = 14 млрд весов в нейросети. Чем больше — тем умнее, но медленнее и тяжелее для железа.

**Ollama** — программа, которая запускает LLM локально. Как Docker, только для нейросетей: скачал модель, запустил, общаешься по HTTP API.

**HTTP API** — способ общения программ через интернет-протокол. Отправляешь POST-запрос с вопросом → получаешь JSON с ответом. Так же работает любой веб-сервис.

---

## Почему именно так

| Решение | Почему |
|---|---|
| CLI, не веб | Проще для демо, не нужен браузер, видно всё в терминале |
| RAG через rag_server | Не дублируем код — сервер уже умеет искать в Qdrant |
| 3 запроса разной сложности | Показываем диапазон: факт → расчёт → список документов |
| Fallback на direct Ollama | Если RAG-сервер упал — утилита всё равно работает |

## Что ещё могло быть

| Вариант | Плюсы | Минусы |
|---|---|---|
| Telegram-бот | Удобен в использовании | Сложнее настроить, не видно в видео |
| Веб-приложение (Flask) | Красивый интерфейс | Больше кода, не суть задания |
| Прямой запрос в Ollama | Проще | Нет RAG, ответы хуже |
| Наш вариант (CLI + RAG) | Чисто, работает, наглядно | Нет GUI |
