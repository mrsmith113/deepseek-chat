# Соглашения проекта yuko-sdk

Правила обязательны для всего кода библиотеки. Проверяются на code review.

## Документирование

- **Каждая публичная функция и метод обязаны иметь docstring.** Публичным
  считается всё, чьё имя не начинается с подчёркивания.
- Docstring на русском языке. Формат: первая строка — краткое описание,
  далее при необходимости разделы «Аргументы:» и «Возвращает:».
- Приватные методы (`_request` и подобные) документировать желательно,
  но не обязательно.

## Именование

- **Имена модулей — `snake_case`**: `api_client.py`, а не `ApiClient.py`
  и не `apiClient.py`.
- Классы — `CamelCase`. Функции, методы и переменные — `snake_case`.
- Константы модуля — `UPPER_SNAKE_CASE`.

## Библиотечный код

- **Никаких `print()` в библиотечном коде.** Диагностику выводить через
  `logging`. `print()` допустим только в примерах и скриптах.
- Никаких сетевых вызовов на импорте модуля.
- Внешних зависимостей нет: только стандартная библиотека.

## Прочее

- Длина строки — до 88 символов.
- Импорты: сначала stdlib, затем модули проекта, разделённые пустой строкой.
- Документация в `docs/` должна соответствовать коду. Меняешь сигнатуру —
  правь `docs/api.md` в том же коммите.
