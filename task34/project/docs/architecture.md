# Архитектура yuko-sdk

Небольшая библиотека для работы с API таможенного брокера Юко.
Зависимостей нет — только стандартная библиотека Python.

## Модули

### `api_client.py`

Транспортный слой. Содержит класс `ApiClient`: хранит `base_url`, токен и
таймаут, выполняет запросы через приватный метод `_request`. Публичные методы —
`get`, `post`, `close` и property `retry_count`. Это единственный модуль,
который знает о деталях протокола; остальные работают только через его
интерфейс.

### `handlers.py`

Слой бизнес-сценариев. Импортирует `ApiClient` напрямую
(`from api_client import ApiClient`). Функции `handle_notification` и
`handle_declaration` принимают готовый клиент и работают с нотификациями ФСБ
и регистрацией деклараций. Функция `make_default_client` собирает клиент с
настройками по умолчанию.

### `reports.py`

Слой отчётности. Импортирует модуль целиком (`import api_client`) и создаёт
клиент как `api_client.ApiClient(...)`. Функция `build_report` собирает данные
отчёта по конфигурации, `export_csv` выгружает готовые строки в CSV-файл.

### `utils.py`

Утилиты без зависимостей от транспорта: `chunked` разбивает последовательность
на куски, `slugify` готовит строку для URL. От `ApiClient` не зависит, поэтому
может использоваться где угодно.

## Поток данных

```
handlers.py ──┐
              ├──> api_client.ApiClient ──> HTTP API
reports.py ───┘

utils.py (независим)
```
