# CLAUDE.md — правила проекта «Обучение БЛМ» (v2.1)

Правила для Claude Code / Cursor / Copilot при работе в этом репозитории.
Эти правила ОБЯЗАТЕЛЬНЫ. При конфликте с общими привычками — приоритет у этого файла.

---

## 1. Что за проект

Учебный репозиторий курса БЛМ (Большие Языковые Модели, Alex Gladkov).
Каждый день курса = отдельный самодостаточный мини-проект в папке `taskN/`.
Общая тема — прикладные LLM-системы: RAG, MCP-инструменты, агенты-оркестраторы,
ассистенты поддержки/разработчика. Прикладная область — ВЭД/таможня (компания Юко).

---

## 2. Стек

| Слой | Чем делаем | Правило |
|---|---|---|
| Язык | Python 3.10+ | Синтаксис `X | None`, `list[dict]` без `typing.List` |
| HTTP | `urllib.request` (stdlib) | НЕ тянуть `requests`/`httpx`/`aiohttp` без реальной нужды |
| Векторный поиск | Qdrant (:6333) или JSON-индекс | Для мелкого таска — JSON-файл, не поднимать сервер |
| Эмбеддинги | GigaEmbeddings (2048 dim) / TF-IDF stdlib | TF-IDF на stdlib, когда эмбеддер не обязателен |
| LLM | DeepSeek API → Ollama (Qwen3) → офлайн-правила | Всегда каскад с деградацией, см. §5 |
| Инструменты модели | MCP-сервер по stdio | stdlib JSON-RPC, без тяжёлых SDK |
| Веб-API (если нужен) | FastAPI | Только когда таск требует HTTP-endpoint |
| venv | `~/rag-env/bin/python` | Там уже стоят все пакеты, новое не ставим без нужды |

Философия: **stdlib-first**. Внешняя зависимость добавляется, только если stdlib
реально не решает задачу. Каждая новая зависимость — в `taskN/requirements.txt`.

## 3. Структура папки таска

```
taskN/
  <module>.py         # по одному модулю на ответственность
  docs/               # markdown-документы для RAG (если есть)
  data/               # json/csv данные (crm.json и т.п.)
  requirements.txt    # только реально используемые пакеты
  run.sh              # запуск одной командой
  README.md           # что это, как запустить
  EXPLANATION.md      # объяснение «как для ребёнка» (для сдачи)
  VIDEO_SCENARIO.md   # сценарий видео (для сдачи)
```

**Одна ответственность на файл.** Каноничная раскладка RAG-таска:
`chunking.py` (нарезка) · `indexer.py` (индекс) · `search.py` (поиск) ·
`assistant.py` (ядро/LLM) · `mcp_*_server.py` (MCP) · `*_tools.py` (доменные функции).
Не сваливать всё в `main.py`.

## 4. Naming conventions

- Модули и функции — `snake_case`: `build_backend`, `chunk_markdown`.
- Классы — `PascalCase`.
- Константы модуля — `UPPER_CASE` вверху файла, с inline-комментом-обоснованием:
  `RAG_MIN_SCORE = 0.02  # ниже этого чанк считаем шумом`.
- Приватные хелперы — с подчёркиванием: `_strip_headings`, `_split_long`.
- Скомпилированные регэкспы — `_NAME_RE`: `_CODE_RE = re.compile(...)`.
- MCP-инструменты — глагол_существительное: `crm_user`, `crm_ticket`, `git_diff`.
- Метаданные чанка — ФИКСИРОВАННАЯ схема dict: `source`, `title`, `date`,
  `section`, `chunk_id`. Не плодить произвольные ключи.

## 5. Обязательные паттерны

### 5.1 LLM-каскад с деградацией
Ответ должен быть содержателен ВСЕГДА. LLM — усилитель, не единственная опора.
Порядок: **DeepSeek API → Ollama → офлайн-ветка (детерминированные правила + RAG)**.
Офлайн-ветка собирает ответ без сети и без ключа. Каждый уровень ловит свою
ошибку и падает на следующий, не роняя процесс.

### 5.2 Заголовок файла
```python
"""<Одна строка: что делает модуль.>

Пайплайн / роль:
  1. ...
  2. ...

Ключевая идея: <почему так, а не иначе>.
"""
from __future__ import annotations
```
Docstring модуля — **на русском**, описывает пайплайн и ключевую идею.

### 5.3 Конфиг через окружение с дефолтом
```python
WEBHOOK = os.getenv("NOTIFY_WEBHOOK", "")   # пусто → печатаем в stdout
```
Секреты — только из env, НИКОГДА не хардкодить в коде.

### 5.4 Визуальные разделители секций
```python
# ------------------------------------------------------------ CRM-контекст --
```

### 5.5 Типы обязательны
Все сигнатуры аннотированы: `def resolve_context(ticket_id: str | None, user_id: str | None) -> tuple[dict | None, dict | None]:`

## 6. Примеры хорошего кода (как ХОТИМ)

**Пример 1 — константа с обоснованием + чистый парс env:**
```python
RAG_TOP_K = 4
RAG_MIN_SCORE = 0.02  # ниже этого чанк считаем шумом
```

**Пример 2 — приватный хелпер, одна мысль, тип на вход/выход:**
```python
def _split_long(text: str, limit: int = MAX_CHARS) -> list[str]:
    """Режет длинный текст по абзацам, не разрывая слова."""
    if len(text) <= limit:
        return [text]
    ...
```

**Пример 3 — роутер: явные маршруты, ранний возврат, лог решения:**
```python
def route_request(user_msg: str, callback_data: str | None = None) -> tuple[str, str | None]:
    if callback_data:
        if callback_data.startswith("alt:"):
            return "ALT_CODES_AGENT", None
    code = extract_code(user_msg)
    if code:
        log.info(f"[Router] CODE_LOOKUP → {code}")
        return "CODE_LOOKUP", code
    return "FULL_PIPELINE", None
```

**Пример 4 — регэксп вынесен в константу, извлечение изолировано:**
```python
_CODE_RE = re.compile(r'\b\d[\d\s]{9,12}\d\b')

def extract_code(msg: str) -> str | None:
    m = _CODE_RE.search(msg)
    if m:
        digits = re.sub(r'\D', '', m.group())
        if len(digits) == 10:
            return digits
    return None
```

**Пример 5 — метаданные чанка по фиксированной схеме:**
```python
chunk = {"source": path.name, "title": doc_title, "date": "",
         "section": section, "chunk_id": f"{path.stem}#{i}", "text": text}
```

## 7. Антипаттерны (ЗАПРЕЩЕНО)

1. **`requests`/`httpx`/`aiohttp`, где хватает `urllib.request`.** Тянуть тяжёлую
   зависимость ради одного POST — нет.
2. **`print()` для отладки в модуле-библиотеке.** Диагностика — через `logging`
   или возвращаемое значение. `print` допустим только в CLI-точке входа/выводе результата.
3. **Хардкод ключей/токенов/URL в коде.** Только `os.getenv`. Секрет в git = провал.
4. **Docstring/комментарии на английском.** Проект русскоязычный — пиши по-русски.
5. **Отсутствие type hints** в сигнатурах публичных функций.
6. **`except:` без типа** (глотает всё, включая KeyboardInterrupt). Ловить конкретное.
7. **Изменяемый аргумент по умолчанию** (`def f(x=[])`). Использовать `None` + инициализацию.
8. **LLM без фолбэка.** Прямой вызов API, который валит весь скрипт при отсутствии
   ключа/сети — нарушение §5.1.
9. **God-file.** Всё в одном `main.py` вместо разбивки по ответственности (§3).
10. **Произвольные ключи метаданных** вместо схемы `source/title/date/section/chunk_id`.
11. **Непойманная доменная ошибка на границе CLI.** Доменные функции бросают
    `crm_tools.CRMError`/`NotFound` при плохом вводе. `main()` ОБЯЗАН ловить их и
    печатать понятное сообщение + `sys.exit(1)`, а не отдавать пользователю traceback.
    (Добавлено в v2.1 по итогам боевой проверки: gen2 падал стектрейсом на битом ticket_id.)

## 8. Шаблон типичного файла

```python
"""<Что делает модуль одной строкой.>

Роль в пайплайне:
  1. <шаг>
  2. <шаг>

Ключевая идея: <обоснование подхода>.
"""
from __future__ import annotations

import os
import json
import urllib.request

# сначала stdlib, потом внутрипроектные импорты
import crm_tools

# ---- Константы -------------------------------------------------------------
TIMEOUT = 10          # сек на HTTP-запрос
DEFAULT_TOP_K = 4     # сколько чанков тянем из RAG


# ---- Публичный API ---------------------------------------------------------
def do_thing(arg: str, opt: int = DEFAULT_TOP_K) -> dict:
    """<Что делает функция.>"""
    ...
    return result


# ---- Приватные хелперы -----------------------------------------------------
def _helper(x: str) -> str:
    ...


# ---- CLI-точка входа -------------------------------------------------------
def main() -> None:
    import sys
    arg = sys.argv[1] if len(sys.argv) > 1 else ""
    print(do_thing(arg))   # print разрешён ТОЛЬКО здесь — вывод результата


if __name__ == "__main__":
    main()
```

Порядок в файле: **docstring → импорты (stdlib → сторонние → внутренние) →
константы → публичный API → приватные хелперы → `main()`/CLI**.

## 9. Слои конфигурации и инструменты

- **Глобальный конфиг:** `~/.claude/CLAUDE.md` — привычки на всех проектах
  (язык, инварианты: не удалять файлы, серверы через подтверждение, секреты не коммитить).
- **Локальный конфиг:** этот файл `GIT/CLAUDE.md` — правила именно репозитория курса.
- **Субагенты:** для крупных тасков — рой (исследователь → исполнитель → валидатор),
  ≤5 параллельно, у каждого одна миссия. Мелкие правки — самому.
- **Скиллы:** `100-клод/скиллы/` — «решаем домашку», «фича», «баг», «новость вэд».
  Триггерятся по фразе, запускают свой пайплайн.

## 10. Definition of Done для таска

- [ ] Код запускается одной командой (`bash run.sh` или `python main.py`)
- [ ] Соответствует §4 naming и §5 паттернам, нет антипаттернов §7
- [ ] `python -m py_compile *.py` — чисто
- [ ] Прогнан на плохом вводе (несуществующий id) — не traceback, а понятная ошибка + exit 1 (см. антипаттерн §7.11)
- [ ] `requirements.txt` содержит только реально нужное
- [ ] Есть `EXPLANATION.md` и `VIDEO_SCENARIO.md`
