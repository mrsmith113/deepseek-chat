# Smoke-сценарии (Level 2) — UI через Playwright

Мишень: `smoke_app.py` — CRUD «Заявки ТН ВЭД». Логин `admin / secret`.
Каждый шаг = скриншот. Раннер (`smoke_runner.py`) протыкивает UI сам и
фиксирует pass/fail + где сломалось.

---

## S1. Логин (happy path)
1. Открыть `/` → форма входа.
2. Ввести логин `admin`, пароль `secret`.
3. Нажать «Войти».
4. **Проверка:** попали на `/items`, виден заголовок «Заявки ТН ВЭД».

## S2. Создать сущность → появилась
1. Предусловие: залогинен, список пуст (счётчик `(0)`).
2. Ввести товар `дрон с камерой`, код `8525893000`.
3. Нажать «Добавить».
4. **Проверка:** в списке есть карточка с товаром и кодом; счётчик `(1)`.

## S3. Создать вторую → счётчик растёт
1. Предусловие: одна заявка уже есть.
2. Добавить `роутер TP-Link` / `8517620009`.
3. **Проверка:** счётчик `(2)`, обе карточки видны.

## S4. Удалить сущность → исчезла
1. Предусловие: две заявки.
2. Нажать «Удалить» у первой.
3. **Проверка:** карточка пропала, счётчик `(1)`, вторая осталась.

## S5. Логин с неверным паролем (negative)
1. Открыть `/` (после logout).
2. Ввести `admin` / `wrong`.
3. Нажать «Войти».
4. **Проверка:** остались на входе, видно сообщение «Неверный логин или пароль»,
   на `/items` без сессии не пускает.

---

## Как гоняется
```bash
.venv/bin/python level2_smoke/smoke_runner.py
```
Раннер сам поднимает сервер, прогоняет S1–S5, кладёт скриншоты в
`level2_smoke/screenshots/`, пишет отчёт `level2_smoke/SMOKE_REPORT.md`.
Если шаг упал — в отчёте шаг, ожидание и скрин момента падения.
