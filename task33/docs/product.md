# Юко-Кабинет — описание продукта

## Что это

«Юко-Кабинет» — SaaS личный кабинет для ВЭД-импортёров электроники и радиоэлектроники. Закрывает цикл: подача заявки на нотификацию ФСБ → отслеживание статуса поставки → хранение и выгрузка документов → интеграция с учётными системами через API.

Веб-приложение: `https://cabinet.notifikat.ru`. API: `https://api.notifikat.ru/v1`.

## Разделы кабинета

| Раздел | Назначение |
|--------|-----------|
| **Дашборд** | Сводка: активные заявки, просроченные задачи, лимиты тарифа, последние события |
| **Заявки** | Создание и ведение заявок на нотификацию ФСБ. Карточка заявки: товар, ТН ВЭД, производитель, криптофункции, комплект документов, статус |
| **Поставки** | Трекинг: номер контейнера/AWB, точки маршрута, статус таможенного оформления, ДТ |
| **Документы** | Хранилище: нотификации (PDF), инвойсы, спецификации, сертификаты. Экспорт ZIP/CSV, поиск по номеру |
| **Интеграции** | API-ключи, вебхуки, журнал доставки вебхуков, SSO (SAML) |
| **Команда** | Пользователи, роли, приглашения, журнал действий (audit log) |
| **Биллинг** | Тариф, счета, акты, способ оплаты, история списаний |
| **Настройки** | Профиль, 2FA, домены компании и их верификация, уведомления |

**Ключевые ограничения (частая причина обращений):**
- **SSO (SAML) доступен только на Enterprise.** При понижении тарифа Enterprise → Pro SSO отключается, все пользователи переводятся на вход по логину/паролю.
- **API-ключи доступны с Pro.** На Free эндпоинты API возвращают 403.
- **Free — 3 заявки в календарный месяц**, счётчик обнуляется 1-го числа. При исчерпании кнопка «Создать заявку» блокируется.
- **SSO работает только после верификации домена** (DNS TXT-запись). Enterprise с невериф. доменом = вход по логину/паролю.

## Роли пользователей

| Роль | Права |
|------|-------|
| **owner** | Всё. Единственный, кто меняет тариф, платит, удаляет организацию, назначает второго owner. Один на организацию (можно передать) |
| **admin** | Управление командой и ролями (кроме owner), API-ключи, вебхуки, SSO и верификация доменов, все заявки и документы. Не видит счета и не меняет тариф |
| **operator** | Создаёт и ведёт заявки, загружает документы, видит поставки. Не управляет командой, ключами, биллингом |
| **viewer** | Только чтение: заявки, поставки, документы. Экспорт разрешён, изменения — нет |

Роль назначается на уровне организации. 2FA может включить любой пользователь себе; admin/owner могут сделать 2FA обязательным для всей организации.

## Статусы заявок

`draft` → `submitted` → `in_review` (проверка комплекта Юко) → `filed` (подана в ФСБ) → `registered` (нотификация зарегистрирована) → `closed`.
Боковые: `needs_info` (запрошены документы, таймер SLA на паузе), `rejected` (отказ ФСБ, доступен разбор причин).

## Интеграции

- **REST API v1** — заявки, документы, поставки, справочники. Аутентификация: заголовок `X-API-Key`. Ключ виден один раз при создании, хранится хешем.
- **Вебхуки** — события `application.status_changed`, `shipment.updated`, `document.added`, `invoice.issued`. POST JSON, подпись `X-Signature` (HMAC-SHA256 по секрету эндпоинта). Ретраи: 5 попыток с экспоненциальной задержкой (1 мин → 16 мин), после — эндпоинт помечается `failing`.
- **SSO SAML 2.0** — Okta, Azure Entra ID, Keycloak, ADFS. Требует Enterprise + верифицированный домен.
- **SCIM 2.0** — автосоздание/деактивация пользователей (Enterprise).
- **1С:Предприятие** — обмен через API (типовая обработка), выгрузка документов и статусов.
- **Telegram-бот уведомлений** — статусы заявок в чат, все тарифы.
