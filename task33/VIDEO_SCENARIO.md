# Сценарий видео — task33: Ассистент поддержки пользователей (~3 мин)

## 0. Вступление (0:00–0:20)

Голос: «День 33. Ассистент первой линии поддержки. Задача — не пересказать
клиенту FAQ, а ответить именно ему: поднять его тикет, его тариф, его флаги
в CRM, свериться с базой знаний и выдать диагноз, шаги и уточняющие вопросы.
Всё офлайн, ноль зависимостей.»

Показать структуру папки:
```bash
ls task33
```

## 1. Из чего собран пайплайн (0:20–0:50)

Голос: «Каждый модуль — одна ответственность.»

Открыть на секунду ASCII-схему в `README.md`, перечислить голосом:
- `crm_tools.py` — карточки клиентов и тикетов из `data/crm.json`.
- `chunking.py` + `search.py` + `indexer.py` — RAG по базе знаний, TF-IDF на stdlib.
- `assistant.py` — ядро: CRM-контекст + RAG + каскад LLM.
- `mcp_crm_server.py` — та же CRM, но как MCP-инструменты для Claude.
- `support.py` — CLI.

## 2. Индекс и MCP (0:50–1:20)

```bash
python3 indexer.py
```
Голос: «База знаний — продукт, FAQ, разбор инцидентов — нарезана на чанки по
секциям. Индекс в JSON, никакой векторной БД.»

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"crm_ticket","arguments":{"ticket_id":"T-1042"}}}' | python3 mcp_crm_server.py
```
Голос: «MCP-сервер на чистом stdlib отдаёт тикет с перепиской. Четыре
инструмента: клиент, тикет, тикеты клиента, поиск. Прописывается в Claude
Desktop одним блоком в конфиге — тогда Claude сам ходит в CRM.»

## 3. Главное: контекст решает (1:20–2:30)

Сначала — вопрос **без тикета**:
```bash
python3 support.py -q "Почему не работает авторизация?"
```
Голос: «Ассистент не знает, кто спрашивает, и честно предлагает статью из FAQ:
шесть типовых причин, проверяйте по списку. Это то, что клиент и сам умеет.»

Теперь — **тот же вопрос с тикетом**:
```bash
python3 support.py --ticket T-1042 -q "Почему не работает авторизация?"
```
Голос: «А теперь ассистент поднял тикет: код SSO-003, область auth. Из него —
клиента: Enterprise, SSO включён, а домен не верифицирован. И ответ другой:
"Игорь, SSO не активен, потому что в DNS домена example-import.ru не
подтверждена TXT-запись". Шесть гипотез схлопнулись в одну причину, названную
по имени. Первым шагом — то, что клиенту нужно сегодня: временно входите по
логину и паролю. Внизу — источники со ссылкой на файл и секцию, контекст
тикета и то, какой генератор сработал.»

Показать пальцем на подвал: `Источники / Контекст / Генератор: offline`.

## 4. Каскад и офлайн (2:30–2:50)

Голос: «Генератор написан честно: offline. Ключа DeepSeek нет, Ollama не
поднята — ответ собрали правила по данным CRM и база знаний. Каскад
DeepSeek → Ollama → офлайн не падает никогда: нейросеть тут усиление, а не
фундамент. Демо не зависит от того, есть ли в аудитории интернет.»

```bash
python3 support.py --demo
```
Голос: «Три кейса: SSO по тикету, клиент на Free упирается в лимит тарифа —
API-ключи только с Pro, и общий вопрос без контекста.»

## 5. Вывод (2:50–3:00)

Голос: «Итог: RAG отвечает на вопрос, а CRM — на вопрос конкретного клиента.
Вместе получается поддержка, а не зачитывание FAQ. Ноль зависимостей,
работает офлайн.»
