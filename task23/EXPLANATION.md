# Объяснение — Task 23: Реранкинг и фильтрация

## Что сделали

Добавили два улучшения к RAG: фильтр качества (threshold + MMR) и умную перефразировку запроса. Это как нанять двух помощников: один убирает ненужные документы со стола, другой переспрашивает тебя "а ты точно это имел в виду?" — и оба вместе помогают найти правильный ответ быстрее.

---

## Ключевые термины

### Cosine similarity (оценка похожести)
Число от 0 до 1 — насколько смысл запроса совпадает со смыслом документа.
- 1.0 = идентичны по смыслу
- 0.7+ = явно релевантно
- < 0.65 = скорее всего шум

**Аналогия**: оценка в школе. Ниже тройки — документ не прошёл отбор.

### Threshold filter (порог отсечения)
Правило: если оценка < 0.65 — выбросить документ. Простой, но эффективный.

**Почему именно 0.65**: эмпирически. Для нашей базы знаний Юко это граница между "видео по теме" и "видео, где слово мелькнуло случайно".

### MMR — Maximum Marginal Relevance
Алгоритм выбора: брать следующий документ, который максимально релевантен И максимально непохож на уже выбранные.

**Аналогия**: составляем команду. Нам нужен лучший игрок — но если три человека делают одно и то же, третий лишний. MMR берёт лучшего, потом лучшего из оставшихся непохожих.

**Альтернативы:**
| Метод | Плюсы | Минусы |
|---|---|---|
| Простой top-K | быстро | дубли, шум |
| Threshold filter | убирает шум | не убирает дубли |
| MMR | разнообразие | нужна функция similarity |
| Cross-encoder | точнее всего | нужна отдельная модель |

### Query Rewriter (переписчик запроса)
LLM (Qwen3) получает вопрос и возвращает расширенный запрос с синонимами и профессиональными терминами.

**Зачем**: пользователь пишет "нотификация ФСБ". В базе знаний это называется "КТС", "шифрование", "ЕАЭС", "уведомительный порядок". Без расширения — часть документов не находится.

**Аналогия**: переводчик с "бытового" на "профессиональный" язык.

### Пайплайн (конвейер)
RAG — это не один шаг, а цепочка:
```
Запрос → Query Rewrite → Поиск → Threshold → MMR → LLM → Ответ
```
Каждый этап улучшает качество независимо.

---

## Почему именно так

- **threshold=0.65**: проверено на нашей коллекции. Для других баз может быть другое значение.
- **top_k=10 → MMR top-5**: берём шире (10), потом отбираем лучшее (5). Если взять сразу 5 — рискуем пропустить хорошие документы, которые чуть дальше по рангу.
- **lambda=0.7 в MMR**: 70% вес на релевантность, 30% на разнообразие. Можно менять.

---

## Итог в одной фразе

Реранкинг — это когда RAG сначала находит кандидатов, а потом перебирает их как рекрутер: убирает слабых, избегает клонов, спрашивает "а вы точно про это?" — и оставляет только лучших.
