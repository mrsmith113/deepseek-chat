# Прогоны двух генераций (доказательства)

Заменяет «скриншоты» — реальный вывод терминала.

## Компиляция
```
$ python3 -m py_compile gen1_notify.py   # → OK
$ python3 -m py_compile gen2_notify.py   # → OK
```
Линтер (pyflakes/ruff) в venv не установлен — проверка ограничена py_compile.

## gen1 (правила v1) — валидный тикет
```
$ python gen1_notify.py T-1042
Тикет:          T-1042
Статус:         open
Тариф / роль:   Enterprise / admin
Код ошибки:     SSO-003
Последнее сообщение: [user] Запись отдавали админам, вроде добавляли. Но сейчас точно не скажу. А как людям работать сегодня?
```

## gen2 (правила v2) — валидный тикет
```
$ python gen2_notify.py T-1042
Тикет T-1042: Не работает авторизация
  Статус:     open
  Клиент:     тариф Enterprise, роль admin
  Код ошибки: SSO-003
  Последнее:  Запись отдавали админам, вроде добавляли. Но сейчас точно не скажу. А как людям работать сегодня?
```

## gen1 — битый тикет (T-9999): аккуратная ошибка
```
$ python gen1_notify.py T-9999
Ошибка CRM: Тикет T-9999 не найден. Есть: T-1042, T-1043, ... T-1049
(exit 1)
```

## gen2 — битый тикет (T-9999): TRACEBACK (регрессия)
```
$ python gen2_notify.py T-9999
Traceback (most recent call last):
  File ".../gen2_notify.py", line 116, in <module>
    main()
  ...
  File ".../task33/crm_tools.py", line 62, in get_ticket
    raise NotFound(f"Тикет {ticket_id} не найден. ...")
crm_tools.NotFound: Тикет T-9999 не найден. Есть: T-1042, ... T-1049
(exit 1)
```

**Разница на скрине:** оба выводят корректную сводку для T-1042 в фирменном стиле,
но на плохом вводе gen1 отдаёт одну строку, а gen2 — стектрейс. Это и породило
правку правил v2 → v2.1 (§7.11).
