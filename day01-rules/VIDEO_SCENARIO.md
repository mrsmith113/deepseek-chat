# Видео-сценарий — День 1: Rules (файл правил проекта)

**Длительность:** 5–7 мин. **Формат:** экран + голос.

---

## 0. Вступление (0:00–0:40)
**Голос:** «День 1 курса — правила проекта. Задача: собрать rules-файл, по которому
ИИ-ассистент пишет код в моём стиле с первого раза. Проверю на боевой задаче: дам
одинаковый промпт двум ассистентам — одному с тонкими правилами, другому с полными,
и сравню результат. Проект — учебный репозиторий по LLM: RAG, MCP, ассистенты
поддержки на Python.»

## 1. Показываю файл правил (0:40–2:00)
**Экран:** открыть `GIT/CLAUDE.md`, проскроллить по разделам.
**Голос, по разделам:**
- «Стек и философия — **stdlib-first**: не тащим `requests`, если хватает `urllib`.»
- «Naming: snake_case, константы UPPER_CASE с комментом-обоснованием, приватные с `_`.»
- «Обязательные паттерны: **LLM-каскад с деградацией** — ответ содержателен всегда,
  DeepSeek → Ollama → офлайн-правила. Docstring модуля на русском с пайплайном.»
- «3–5 примеров хорошего кода — прямо из моих файлов, как я ХОЧУ чтобы писали.»
- «Антипаттерны — что запрещено: requests вместо urllib, print для отладки, хардкод
  ключей, english-docstring, bare except, LLM без фолбэка.»
- «Шаблон типичного файла — порядок: docstring → импорты → константы → API →
  хелперы → main.»
- «Слои конфига: глобальный `~/.claude/CLAUDE.md`, локальный этот, субагенты, скиллы.»

## 2. Боевая задача — один промпт (2:00–2:40)
**Экран:** показать текст промпта из `DIFF.md`.
**Голос:** «Один промпт, без переписок: новый сервис `notify.py` — сводка тикета на
webhook, при пустом URL печать в stdout. Запускаю ДВЕ генерации параллельно:
gen1 видит только тонкие правила v1, gen2 — полные v2.»

## 3. Смотрим результат (2:40–4:00)
**Экран:** `gen1_notify.py` и `gen2_notify.py` рядом.
**Голос:** «Оба скомпилировались, оба в стиле проекта — потому что модель сильная и
прочитала соседний файл `crm_tools.py`, скопировала конвенции. Но у v2 детали
осмысленнее: возвращает канал доставки, тянет subject тикета, безопасно достаёт
юзера через `_safe_user`.»
**Команда:**
```
python3 gen1_notify.py T-1042
python3 gen2_notify.py T-1042
```
Показать одинаково корректные сводки.

## 4. Ключевой момент — правила ≠ корректность (4:00–5:20)
**Команда:**
```
python3 gen1_notify.py T-9999   # аккуратная ошибка, exit 1
python3 gen2_notify.py T-9999   # TRACEBACK
```
**Голос:** «А теперь битый id. Тонкий gen1 — аккуратная ошибка. „Правильный по
правилам“ gen2 — стектрейс! Он обернул только запрос юзера, а тикет — нет. Вывод:
**rules-файл гарантирует стиль, но не корректность.** Тесты никто не отменял.»

## 5. Итерация правил v2 → v2.1 (5:20–6:20)
**Экран:** показать добавленный §7.11 и пункт DoD в `CLAUDE.md`.
**Голос:** «Дыру закрываю правилом: „лови доменные ошибки на границе CLI — понятное
сообщение и exit 1, не traceback“. Плюс пункт Definition of Done — прогнать на плохом
вводе. Это и есть усиление правил по результатам боевой проверки.»

## 6. Вывод (6:20–7:00)
**Голос:** «Что сильнее всего повлияло на качество: первое — живой эталонный файл в
репозитории, его модель копирует лучше любых слов. Второе — явные примеры и
антипаттерны. Третье — требование фолбэка. Правила v2.1 + два дифа генераций —
в папке `day01-rules`. Готово.»
