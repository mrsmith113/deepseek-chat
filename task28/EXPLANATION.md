# Объяснение: День 28 — Локальная LLM + RAG

## Что мы сделали — одним абзацем

Мы взяли базу знаний из 351 YouTube-видео (уже проиндексированную на прошлой неделе), подключили к ней локальную языковую модель Qwen3 14B, и получили систему, которая отвечает на вопросы по ВЭД — **полностью без интернета**. Потом сравнили её с облачным DeepSeek: кто быстрее, кто умнее, кто надёжнее.

---

## Каждый термин — своими словами

### RAG (Retrieval-Augmented Generation)
**Буквально:** "Генерация с подкреплением поиском"

**Как объяснить ребёнку:** Представь, что тебе задали вопрос по истории. Вместо того чтобы вспоминать из головы, ты сначала открываешь нужные страницы учебника, читаешь их — и только потом отвечаешь. RAG делает то же самое: сначала ищет нужные куски текста, потом формулирует ответ на их основе.

**Зачем:** Без RAG LLM отвечает из "памяти" — которую нельзя обновить. С RAG — из актуальной базы знаний.

---

### Embedding (эмбеддинг / векторное представление)
**Буквально:** "Вложение" — преобразование текста в список чисел

**Как объяснить:** Представь что каждое слово — точка в комнате. Близкие по смыслу слова — рядом. "Таможня" и "оформление" стоят близко, "таможня" и "рецепт" — далеко. GigaEmbeddings превращает целые абзацы в такие точки — и мы ищем ближайшие к вопросу.

**Наш инструмент:** `ai-sage/Giga-Embeddings-instruct` — русскоязычная модель, 2048 чисел на один текст.

---

### Qdrant — векторная база данных
**Буквально:** База данных, которая хранит векторы и умеет быстро находить похожие.

**Как объяснить:** Обычная база данных ищет "где написано слово X". Qdrant ищет "что похоже по смыслу на Y". Это как разница между поиском по слову в книге (Ctrl+F) и поиском "найди абзацы про похожую тему".

**Наша коллекция:** `youtube_rag` — 1792 вектора из транскриптов 351 видео канала @youkogroup.

---

### Ollama — локальный сервер для LLM
**Буквально:** Программа, которая запускает большие языковые модели на вашем компьютере.

**Как объяснить:** Это как ChatGPT, только работает на вашем компьютере, не отправляет данные никуда, и бесплатный. Минус — нужна мощная видеокарта или много RAM.

**Наша модель:** `qwen3:14b` — 14 миллиардов параметров, качество примерно как GPT-3.5.

---

### DeepSeek — облачная LLM
**Буквально:** Китайская языковая модель, доступная через API как ChatGPT.

**Как объяснить:** Вы отправляете запрос на сервер в интернете, получаете ответ. Быстро, умно, но данные уходят во внешний сервис.

**Зачем сравниваем:** Хотим понять — стоит ли трудиться с локальной установкой, или проще платить за API?

---

## Почему именно так

| Решение | Альтернатива | Почему выбрали |
|---------|-------------|----------------|
| GigaEmbeddings | OpenAI embeddings, E5 | Лучшее качество на русском, уже установлена |
| Qdrant | FAISS, JSON-индекс | Полноценный сервер, умеет фильтры, уже запущен |
| Qwen3 14B | LLaMA, Mistral | Лучший на русском среди открытых моделей |
| DeepSeek для сравнения | GPT-4, Claude | Дешевле, API-совместимый, хорошее качество |

## Что ещё могло быть

### Вариант А — без Qdrant (JSON-индекс, как в task21)
| Плюсы | Минусы |
|-------|--------|
| Проще (нет сервиса) | Медленнее при > 10к чанков |
| Нет зависимостей | Нельзя обновлять на ходу |

### Вариант Б — через RAG-сервер (:8000, как в task27)
| Плюсы | Минусы |
|-------|--------|
| Готовый API | Ещё один сервис для поддержки |
| Легко интегрировать | Скрывает детали (плохо для обучения) |

### Вариант В — без RAG (прямой вопрос в LLM)
| Плюсы | Минусы |
|-------|--------|
| Проще | Не знает нашу базу знаний |
| Быстрее | Может галлюцинировать |

**Мы выбрали:** прямое подключение к Qdrant + Ollama — максимально прозрачно для учёбы.
