# Объяснение task32 — как для ребёнка

## Что мы сделали (одним абзацем)

Представь, что в мастерскую приносят починенную игрушку, и опытный мастер
быстро осматривает: не торчит ли гвоздь, надёжно ли держится колесо, красиво ли
покрашено. Мы сделали такого «мастера», но для кода. Когда программист приносит
изменения (Pull Request), наш помощник сам смотрит, что именно поменяли,
сверяется с правилами и документацией проекта, и пишет отзыв: где возможна ошибка,
где криво устроено, что стоит улучшить. И делает это сам, без человека.

## Разбор терминов (своими словами + пример из жизни)

| Термин | Простыми словами | Пример из жизни |
|---|---|---|
| **Pull Request (PR)** | Заявка «я поменял вот это, гляньте» | Ученик сдаёт тетрадь на проверку |
| **diff** | Список: что убрали, что дописали | Красная ручка учителя: зачёркнуто/добавлено |
| **изменённые файлы** | Какие страницы тетради трогали | «Смотри только страницы 3 и 7» |
| **RAG** | Помощник сначала читает справочник, потом отвечает | Врач заглядывает в карту пациента перед диагнозом |
| **эвристики** | Простые правила «если увидел X — это плохо» | «Если провод оголён — опасно», без раздумий |
| **LLM** | Умная нейросеть, пишет отзыв словами | Опытный наставник объясняет, что не так |
| **GitHub Action** | Робот, который сам запускается на каждую заявку | Автоматическая мойка: заехал — помыли |
| **TF-IDF** | Способ найти похожий кусок текста по словам | Ищешь в оглавлении главу по ключевому слову |

## Почему сделали именно так

- **Разбили на маленькие кубики** (парсер, эвристики, поиск, ревьюер). Один
  кубик — одно дело. Легко чинить и понимать, как коробка с отдельными ящичками,
  а не свалка.
- **Сделали, чтобы работало без интернета.** Умная нейросеть (LLM) не всегда под
  рукой: то ключа нет, то сервер выключен. Поэтому есть простые правила
  (эвристики), которые всегда дадут отзыв. Нейросеть — приятный бонус сверху,
  а не единственная опора. Как фонарик со свечкой про запас.
- **RAG по коду И документации.** Мастер смотрит не только на саму деталь, но и
  на чертёж всего изделия. Так отзыв точнее: помощник знает правила проекта.
- **Переиспользовали прошлый таск (task31.1).** Поиск и работу с git уже писали
  раньше — не стали делать заново. Зачем изобретать колесо.

## Что ещё могло быть (альтернативы)

| Вариант | Плюс | Минус |
|---|---|---|
| Только нейросеть (LLM) | Красивый живой отзыв | Падает без сети/ключа, стоит денег, может выдумать |
| Только эвристики (наш fallback) | Быстро, бесплатно, надёжно | Не понимает смысл, ловит лишь шаблоны |
| Готовый линтер (flake8/eslint) | Проверено временем | Не знает контекст проекта, не пишет человеческий отзыв |
| **Наш гибрид (эвристики + RAG + LLM)** | Работает всегда, учитывает контекст | Чуть больше кода собрать |
| Векторная БД Qdrant вместо JSON | Масштаб на миллионы файлов | Надо поднимать сервер; для одного репо избыточно |

## Итог одной фразой

Робот-рецензент: сам берёт изменения в коде, сверяется с правилами и
документацией проекта и пишет понятный отзыв — где баг, где криво, что улучшить —
причём работает даже без нейросети.
