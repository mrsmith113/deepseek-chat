# Сценарий видео — task32: Автоматизация ревью кода (5–7 мин)

## 0. Вступление (0:00–0:40)
Голос: «День 32. Задача — сделать так, чтобы новый Pull Request автоматически
проходил AI-ревью. Ассистент получает diff, использует RAG по документации и
коду, и возвращает три вещи: потенциальные баги, архитектурные проблемы,
рекомендации. Всё работает офлайн, а в CI — как GitHub Action.»

Показать на экране структуру папки:
```bash
ls task32
```

## 1. Из чего собран пайплайн (0:40–2:00)
Голос: «Пять кубиков, каждый — одна ответственность.»
Открыть по очереди, коротко:
- `diff_parser.py` — парсит unified diff, достаёт добавленные строки.
- `heuristics.py` — статический анализатор: секреты, eval, широкий except…
- `search.py` + `chunking.py` — RAG над **доками и кодом** (TF-IDF, офлайн).
- `reviewer.py` — склеивает всё + LLM-каскад DeepSeek→Ollama→без LLM.
- `.github/workflows/ai-review.yml` — запуск на PR.

## 2. Строим индекс (2:00–2:40)
```bash
python3 indexer.py
```
Голос: «RAG-индекс собран из README, docs и всех .py — 58 чанков,
документация плюс код. Это контекст, с которым ассистент сверяет изменения.»

## 3. Ревьюим PR (2:40–4:30)
Показать демо-diff с зашитыми проблемами:
```bash
cat samples/sample_pr.diff
```
Голос: «Тут захардкожен ключ, eval на входных данных, голый except,
мутабельный дефолт — типичные грабли.»

Запуск:
```bash
python3 review.py --diff samples/sample_pr.diff
```
Показать вывод: три раздела, находки с файлом и строкой, RAG-контекст в конце.
Голос: «Ассистент нашёл секрет, инъекцию через eval, широкий except,
сравнение с None — всё с точным файлом и номером строки.»

## 4. Как это в CI (4:30–6:00)
Открыть `.github/workflows/ai-review.yml`.
Голос: «На каждый pull request Action берёт git diff BASE...HEAD, гоняет тот же
review.py и постит результат комментарием в PR. Ключ DeepSeek — опционально
через secrets; без него ревью соберут эвристики и RAG. То есть пайплайн не
падает, если LLM недоступна.»

## 5. Вывод (6:00–6:40)
Голос: «Итог: автоматическое AI-ревью кода. PR → diff → RAG по докам и коду →
текст ревью с багами, архитектурой и рекомендациями. Ноль внешних зависимостей,
работает и локально, и в GitHub Actions.»
