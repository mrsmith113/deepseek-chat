# Сценарий видео — task32: Ассистент разработчика

**Длительность:** 5–7 минут. **Цель:** показать ассистента, который понимает
проект — через RAG над документацией + MCP-доступ к git + команду `/help`.

---

## 0. Вступление (0:00–0:40)

Говорить голосом:
> «День 32. Делаю ассистента разработчика. Он берёт документацию проекта —
> README и папку docs — кладёт в RAG, а через MCP получает живой контекст git:
> текущую ветку, файлы, diff. Команда /help отвечает на вопросы о проекте,
> объединяя документацию и состояние репозитория. Всё запускается на голом
> python3, без внешних сервисов.»

Показать структуру папки:
```bash
ls task32
ls task32/docs
```

---

## 1. Документация в RAG (0:40–2:00)

Говорить: «Сначала документация. Вот README и три файла в docs — архитектура,
карта тасков, описание API и команд. Их и индексирует ассистент.»

```bash
cd task32
python3 indexer.py
```
Показать вывод: `24 чанка из 4 документов`.

Объяснить: «`chunking.py` режет markdown по заголовкам-секциям, к каждому чанку
— метаданные: файл, заголовок, секция, id. `indexer.py` складывает всё в
`index.json` — векторная БД не нужна.»

Показать поиск:
```bash
python3 search.py
```
Говорить: «Поиск на TF-IDF, чистый stdlib — работает офлайн. Если нужен dense —
ставится sentence-transformers и включается через EMB_BACKEND=dense.»

---

## 2. MCP — доступ к git (2:00–3:30)

Говорить: «Теперь MCP. Сервер `mcp_server.py` говорит по протоколу Model
Context Protocol через stdio. Даёт три инструмента: git_branch, git_files,
git_diff.»

Список инструментов:
```bash
printf '%s\n' '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}' | python3 mcp_server.py
```

Вызов git_branch (минимум задания):
```bash
printf '%s\n' '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"git_branch","arguments":{}}}' | python3 mcp_server.py
```
Показать ответ: `"text": "master"`.

Говорить: «Ассистент вызывает этот сервер как настоящий MCP-клиент — запускает
процесс, шлёт initialize, потом tools/call. Логика git — в `git_tools.py`,
её переиспользуют и сервер, и ассистент.»

---

## 3. Команда /help (3:30–5:30)

Говорить: «Главное — команда /help. Задаём вопрос о проекте.»

```bash
python3 assistant.py "как устроен RAG и какие режимы у CLI?"
```

Разобрать вывод по шагам:
1. `[индекс: 24 чанка, backend=tfidf]` — загрузился RAG.
2. Ответ — топ-фрагменты из документации с указанием источника.
3. Git-контекст внизу — ветка master + свежий diff (через MCP).

Показать интерактив:
```bash
python3 assistant.py
# /branch
# /help как коммитить задание?
# выход
```

Сказать про LLM: «Если задать `DEEPSEEK_API_KEY` — ответ генерит DeepSeek по
найденному контексту. Нет ключа — работает Ollama. Нет и его — extractive-режим
собирает ответ из фрагментов. Ассистент отвечает всегда.»
