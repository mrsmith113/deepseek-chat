# Карта заданий (taskN)

Репозиторий устроен по принципу «одно задание — одна папка/файл». Ниже —
ориентир по ключевым таскам.

## Ранние таски (один файл в корне)

- `cli.py` — CLI-чат с DeepSeek, режимы free/strict/nano, история диалога.
- `bot.py` — Telegram-бот поверх DeepSeek.
- `task3.py`…`task11.py` — промпт-инжиниринг, функции, память диалога,
  структурированный вывод, ранние эксперименты.

## RAG-таски (папки)

- `task21` — базовый RAG: чанкинг + индексатор + сравнение (JSON-индекс без сервера).
- `task22` — оценка качества RAG (eval), настройка `num_predict`.
- `task23` — реранкинг: threshold-фильтр, MMR, переписывание запроса.
- `task24` — RAG с цитатами (ссылки на источник в ответе).
- `task25` — RAG-чат с памятью в Telegram (GigaEmbeddings + Qdrant + DeepSeek).
- `task26`+`task27` — локальная LLM (Ollama/Qwen3) + RAG CLI-утилита.
- `task28` — локальный RAG офлайн.
- `task29` — бенчмарк и оптимизация RAG.

## MCP и агенты

- `task17` — MCP-сервер к CouchDB (`api1.notifikatai.ru/mcp`), systemd + nginx.
- `task18` — challenge-bot + challenge-mcp.
- `task30` — Claude-прокси + харнесс с tool-calling.
- `task31` — агентная платформа: YAML-профили агентов.
- `task32` — ассистент разработчика: RAG над docs + MCP git-контекст + `/help`.

## Правила

- Ветка разработки — `master`.
- Каждый таск запускается одной командой.
- Новый код кладём в `taskN/`, не в старые папки.
