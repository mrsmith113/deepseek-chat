# Сценарий видео — Task 21: Индексация документов

**Продолжительность:** ~5-7 минут  
**Формат:** скринкаст терминал + объяснение голосом  
**Терминал:** WSL (bash) — не PowerShell. Открыть: Win+R → `wsl` или вкладка WSL в Windows Terminal.

---

## Структура видео

### 1. Вступление (30 сек)
> "День 21 — индексация документов. Мы уже развернули RAG-агент с 351 видео канала @youkogroup. Сегодня покажу, как работает сам механизм — на примере 10 документов, двух стратегий чанкинга и сравнения качества поиска."

### 2. Показываем документы (1 мин)
```bash
ls documents/
wc -l documents/*.md | tail -1   # общее кол-во строк
```
> "10 статей о ВЭД — нотификации, переводы в Китай, временный ввоз. Суммарно — больше 30 страниц текста."

### 3. Две стратегии чанкинга (1.5 мин)

**Запускаем indexer.py и объясняем по ходу:**
```bash
~/rag-env/bin/python indexer.py
```

Пока идёт индексация — объясняем:

> "Стратегия 1 — Fixed: берём 500 символов, откатываемся на 100 назад, берём следующие 500. Просто, предсказуемо, но может разрезать фразу посередине.

> Стратегия 2 — Struct: режем по заголовкам ## документа. Каждый раздел — один чанк. Размер непостоянный, но смысл всегда целый."

После завершения:
```
index_fixed.json  — 55 чанков (avg 453 chars)
index_struct.json — 38 чанков (avg 537 chars)
```

### 4. Ключевой момент — compare.py (2.5 мин)
```bash
~/rag-env/bin/python compare.py
```

**Останавливаемся на первом запросе** "как перевести деньги в Китай":

> "Смотрим: Fixed нашёл чанк с score 0.64 — но текст начинается с '...аны), теперь насчитывается...' — это середина предложения, контекст потерян.

> Struct нашёл score 0.69 — целый раздел '## Итог: 6 рабочих способов'. Видно откуда, что это, смысл читается сразу.

> Разница небольшая — 0.04, но качество читаемости принципиально разное."

**Итоговая таблица победителей:**
```
Запрос 1: Struct ✅ (Δ=0.046)
Запрос 2: Struct ✅ (Δ=0.028)  
Запрос 3: Struct ✅ (Δ=0.014)
```

### 5. Заключение (30 сек)
> "Вывод: для структурированных документов с заголовками — используйте struct-чанкинг. Fixed хорош для неструктурированного текста (PDF без разметки, raw-транскрипты).

> Индекс — JSON-файл с эмбеддингами и метаданными. В продакшне заменяется на Qdrant или FAISS, но принцип тот же.

> Код на GitHub в task21/."
