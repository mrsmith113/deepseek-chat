# День 25. Мини-чат с RAG + памятью (production-like)

## Задание

Реализовать мини-чат, который:
- хранит историю диалога
- при каждом вопросе ищет контекст через RAG
- отвечает с учётом найденной информации
- всегда выводит источники
- хранит "память задачи": цель, термины, что уточнено

Проверить на 2 сценариях по 10-15 сообщений.

---

## Реализация

**Платформа:** Telegram-бот (`@Deepseek_youko_bot`)  
**Скрипт:** `/home/stashome/rag-agent/rag_tg_bot.py`  
**Сервис:** `systemctl --user status rag-tg-bot`  
**Логи:** `/home/stashome/rag-agent/rag_tg_bot.log`

### Стек
| Компонент | Решение |
|---|---|
| Интерфейс | Telegram Bot API (long polling) |
| Эмбеддинги | GigaEmbeddings (`ai-sage/Giga-Embeddings-instruct`) |
| Векторная БД | Qdrant `tnved_rag` (60 000+ точек) |
| Генерация | DeepSeek API (`deepseek-chat`) |
| Python | `/home/stashome/rag-env/bin/python3` |

### Архитектура

```
Пользователь → Telegram
    ↓ сообщение
rag_tg_bot.py
    ├── GigaEmbeddings.encode(вопрос)
    ├── Qdrant.search(top_k=5) → чанки + источники
    ├── История диалога (последние 10 сообщений)
    ├── Память задачи (цель / термины / что уточнено)
    └── DeepSeek(system + контекст + история + вопрос)
    ↓ ответ + источники [1][2][3]
Пользователь ← Telegram
```

### Память задачи
Каждые 3 хода DeepSeek анализирует диалог и обновляет структуру:
```json
{
  "goal": "цель диалога",
  "terms": ["зафиксированные термины"],
  "clarified": ["что уже уточнено"]
}
```
Команда `/memory` — посмотреть текущую память.

---

## Команды бота

| Команда         | Действие                        |
| --------------- | ------------------------------- |
| `/start`        | Приветствие                     |
| `/memory`       | Показать память задачи          |
| `/clear`        | Сбросить историю и память       |
| `<любой текст>` | RAG-поиск + ответ с источниками |
