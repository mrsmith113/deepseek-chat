# Видео-сценарий — День 2: Profiles (профили ассистента)

**Длительность:** 6–8 мин. **Формат:** экран + голос.

---

## 0. Вступление (0:00–0:45)
**Голос:** «День 2 — профили. День 1 научил ассистента писать код в моём стиле.
Но стиль ≠ дисциплина работы. Сегодня делаю три профиля поведения: **Bug Fix**,
**Research** и третьим — **Code Review**. Каждый — это системная рамка: что ассистент
ДОЛЖЕН делать, что ему ЗАПРЕЩЕНО, и в каком формате отвечать. Проверю все три на
реальных задачах из своего репозитория — один запуск, рабочий результат.»

## 1. Показываю файл профилей (0:45–2:15)
**Экран:** открыть `day02-profiles/PROFILES.md`, скролл по трём профилям.
**Голос, по каждому:**
- «Bug Fix. Промпт: воспроизведи → найди корень → минимальный фикс → докажи, что не
  сломал. Запрещено: лечить симптом, игнорить проверку. Формат: баг / воспроизведение /
  причина / фикс / проверка / что не проверил / близнецы.»
- «Research. Промпт: исследуй кодовую базу, ничего не меняй, якорь `файл:строка` на
  каждый факт. Запрещено: править код, цитировать неоткрытый файл. Формат: файлы /
  связи / разбор / вывод / уверенность.»
- «Review. Промпт: найди реальные дефекты, дай точечный фикс, не хвали. Одна находка —
  одна строка. Ранжирование 🔴🟡🔵.»

## 2. Тест Bug Fix — реальный баг (2:15–4:00)
**Экран:** `BUGFIX_RUN.md`.
**Голос:** «Боевой баг: в task33 Ollama будто не видит весь RAG-контекст, а в task34
всё ок. Профиль сработал правильно: не полез гадать про timeout, а воспроизвёл —
промпт до 4000+ токенов, а запрос в Ollama шёл БЕЗ `num_ctx`. Дефолт qwen3 — 4k,
всё сверх окна режется молча. И главное — профиль велел сверить с эталоном: рядом
`task34/llm.py` уже с `num_ctx`. Значит это не баг, а непортированный фикс.»
**Команда:**
```
git -C GIT diff task33/assistant.py     # показать применённый фикс
python3 -m py_compile GIT/task33/assistant.py && echo OK
```

## 3. Тест Research — вопрос по проекту (4:00–5:15)
**Экран:** `RESEARCH_RUN.md`.
**Голос:** «Вопрос: как устроен LLM-каскад в task33. Профиль дал структуру со
ссылками на строки: `answer():351` — переменная `body`, DeepSeek → Ollama → офлайн,
переход только на None, офлайн-ветка None не возвращает — дно каскада. Каждый тезис
кликается в код. И честная секция уверенности: живой DeepSeek — гипотеза, не запускал.»

## 4. Тест Review — третий профиль (5:15–6:15)
**Экран:** `REVIEW_RUN.md`.
**Голос:** «Ревью того же файла после фикса. Плотно, одной строкой на находку: два
🟡 — широкий `except Exception` против §7.6, пять 🔵 — что сделано правильно. Вердикт:
мержить, блокеров нет. Без воды и без „перепиши всё“.»

## 5. Что пришлось доработать (6:15–7:20)
**Экран:** `ITERATIONS.md`.
**Голос:** «Ключевое по заданию — доработка после первой попытки. v1 Bug Fix нашёл
СИМПТОМ — предложил крутить timeout. Добавил в профиль обязательное воспроизведение
и сверку с эталоном — со второй версией диагноз стал верным. v1 Research отвечал по
памяти без якорей — добавил жёсткое `файл:строка` на каждый факт и секцию уверенности.
v1 Review хвалил и раздувал скоуп — ужал до одной строки на находку. Вывод: правила
дают стиль, профили дают проверяемость.»

## 6. Вывод (7:20–8:00)
**Голос:** «Три профиля: Bug Fix воспроизводит прежде чем чинить, Research якорит
каждый факт в код, Review режет по одной строке. Все три прогнаны на реальных задачах
с первого запуска. Файлы — в `day02-profiles`. Готово.»

